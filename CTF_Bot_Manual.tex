\section{Introduction}
cyberBOT 2.0 is a next-generation CTF engine designed for high-stakes digital competitions. This manual provides the total operational protocols for administrators. If you are reading this, you are the designated Overseer of the competition.

\section{System Architecture}
\subsection{Core Framework}
The bot is built on \texttt{discord.py} with an asynchronous core.
\begin{itemize}
    \item \textbf{Asynchronous DB:} Uses \texttt{aiosqlite} for non-blocking I/O.
    \item \textbf{Write-Ahead Logging (WAL):} Database concurrency is optimized for high-volume flag submissions.
    \item \textbf{Cog System:} Features are modularized into \texttt{Admin} and \texttt{Player} logic.
\end{itemize}

\subsection{Database Persistence}
The system state is stored in \texttt{bot.db}. 
\textbf{CRITICAL:} Always perform a \texttt{/export} before making structural server changes.

\section{Administrator Protocols}
\subsection{Initial Synchronization}
Before the event, you must link the bot to the server's neural network:
\begin{enumerate}
    \item Run \texttt{/setup}: Assign the Leaderboard, Logs, and General channels.
    \item Run \texttt{/set\_rank\_role}: Map point thresholds to server roles. This enables the auto-promotion engine.
\end{enumerate}

\subsection{Mission Deployment}
Missions are created in two stages:
\begin{enumerate}
    \item \textbf{Stage 1 (Create):} Use \texttt{/create} to register the mission ID, flag, points, and category. You must use one of the 7 official dropdown categories (WEB, CRYPTO, etc.).
    \item \textbf{Stage 2 (Post):} Use \texttt{/post} to publish the mission. This allows for immediate posting or scheduling for a future timestamp.
\end{enumerate}

\subsection{Championship Logic}
The bot monitors the Leaderboard every 2 minutes. When a new \#1 player emerges:
\begin{itemize}
    \item The bot removes the \textbf{Champion's Belt} (role) from the old leader.
    \item It assigns the role to the new leader.
    \item An automated alert is broadcast to the General channel: \textit{``Agent X has stolen the belt!''}
\end{itemize}

\section{Emergency Recovery}
\subsection{Database Restore}
In the event of system corruption:
\begin{enumerate}
    \item Locate your latest \texttt{bot.db} backup.
    \item Use \texttt{/import}: Upload the backup file.
    \item The bot will live-swap the database and re-initialize all triggers instantly.
\end{enumerate}

\subsection{Refund Logic}
If a mission is deleted via \texttt{/delete}, the bot executes a recursive refund. Every Agent who spent points on hints for that mission will have their points automatically restored to their score.
