\chapter{System Introduction}
% ============================================================

\textbf{CyberBOT} is a fully asynchronous Capture The Flag engine for Discord, rebuilt with a
dossier-style intelligence profile system, automated role progression, a competitive hint economy,
and a complete administrative suite. It is designed to run an entire CTF season with zero manual
intervention once configured.

\section{What is New in CyberBOT}

\begin{itemize}
    \item \textbf{Dossier Profile Cards:} Completely redesigned 1800$\times$700 Agent ID cards with dark
          charcoal folder aesthetics, neon HUD borders, CLASSIFIED watermark, and dynamic clearance
          progression bars.
    \item \textbf{Specialization Tags:} HUD boxes showing the categories of the last 3 challenges solved.
    \item \textbf{Clearance Progression:} A progress bar showing advancement toward the next rank role.
    \item \textbf{Max Clearance State:} When the highest rank role is achieved the progress bar is replaced
          with a MAX CLEARANCE LEVEL ATTAINED banner.
    \item \textbf{The Architect Easter Egg:} Running \cmd{/profile} on the bot itself reveals a unique
          \textbf{[ROOT]} profile with SYSTEM CORE badge, infinity score, and the ARCHITECT OF THE
          SIMULATION signature.
    \item \textbf{Dropdown Categories:} Challenge categories are enforced via slash command dropdowns,
          ensuring clean specialization tag rendering.
    \item \textbf{Belt Steal Announcements:} When the \#1 ranking changes, the bot broadcasts a live
          alert to the General channel.
\end{itemize}

\section{System Requirements}

\begin{itemize}
    \item \textbf{Python 3.10+} --- Required for modern \texttt{asyncio} features.
    \item \textbf{Discord Bot Token} --- From the Discord Developer Portal.
    \item \textbf{Gateway Intents} --- Server Members, Presence, and Message Content must be enabled
          in the Developer Portal.
    \item \textbf{Bot Permissions} --- Send Messages, Embed Links, Attach Files, Manage Roles,
          Read Message History.
    \item \textbf{font.ttf} --- Must be present in the root directory for profile card rendering.
\end{itemize}

\section{Championship Belt Logic}

cyberBOT monitors standings every 2 minutes. When the \#1 player changes:

\begin{enumerate}
    \item The Champion role is stripped from the previous holder.
    \item The Champion role is granted to the new leader.
    \item An automated Belt Stolen alert is broadcast to the configured General channel.
\end{enumerate}

\chapter{Anti-Deletion Watchdog}
% ============================================================

A background task polls every 60 seconds. For every active posted challenge, it verifies that both
the mission embed and its file attachment message still exist. If either is missing, full recovery
is triggered automatically.

\begin{diagram}{Diagram 6.1 --- Watchdog Heartbeat Cycle}

    \node (clean)   [danger,  below=1.8cm of missing]         {Delete \\ fragments};
    \node (repost)  [ok,      left=1.5cm of clean]           {Re-post embed \\ + file};
    \node (sync)    [block,   above=1.6cm of repost]          {Update msg\_id \\ in DB};

    \draw [arrowcyan]  (poll)    -- (fetch);
    \draw [arrowcyan]  (fetch)   -- (missing);
    \draw [arrowgreen] (missing) -- node[above, font=\tiny]{No}  (ok);
    \draw [arrowred]   (missing) -- node[right, font=\tiny]{Yes} (clean);

The watchdog also handles expiry: when \texttt{end\_time} is reached the embed is updated to show
\textbf{[Expired]}, the Submit button is disabled, and the local file is deleted from storage.

\section{Millisecond Tie-Breaking}

Timestamps are stored as \texttt{REAL} (Python \texttt{float}), giving sub-millisecond precision.
When two players share the same point total, the one who reached that total \textit{first} ranks
higher. The leaderboard SQL sorts by \texttt{points DESC, last\_solve\_timestamp ASC}.

\section{Hint Economy}

\begin{itemize}
    \item Hints are purchased with points via a dropdown selector on the mission card.
    \item Once purchased, a hint is permanently unlocked for that player at no further cost.
    \item \cmd{/remove\_hint} triggers automatic refunds to all buyers.
    \item \cmd{/delete} refunds all hint costs before purging the challenge.
\end{itemize}

\section{Collusion Detection}

If the same flag is solved by two different players within 60 seconds of each other, a collusion
alert is logged to the Challenge Logs channel with both usernames and the exact time delta. The
solve is still recorded; the alert is for admin review.

\section{/profile}

Renders and sends the Agent dossier card as a PNG. Optionally accepts a \param{member} mention
to view another player's card.

Interactive paginated embed (10 players per page) with Previous/Next navigation. Sorted by points
descending with timestamp tie-breaking. A \textbf{live leaderboard message} is also maintained
in the configured channel, auto-refreshing every 2 minutes and on every score change.

Complete command reference. Administrators see the full suite including mission control and
maintenance commands. Players see agent-facing commands only. Ephemeral.

\section{/about}

Displays comprehensive system specifications and development credits for the Lead Architect
and System Engineer. Features a live Satellite Uplink diagnostic showing real-time
network latency and encryption protocols. This command is delivered ephemerally to
maintain operational security.

\chapter{Administration Reference}

\section{Mission Control}

\begin{description}[leftmargin=0pt, itemsep=0.4em]
    \item[\cmd{/create} \param{challenge\_id points flag category} [\param{image\_url}]]
    Register a challenge in draft mode.

    \item[\cmd{/post} \param{challenge\_id start\_time end\_time} [\param{channel description connection\_info file}]]
    Deploy or schedule. Time format: \texttt{DD/MM HH:MM}.

    \item[\cmd{/edit} \param{challenge\_id} [\param{new\_id points flag category image\_url description connection\_info start\_time end\_time}]]
    Modify any attribute. Live embed syncs immediately. Renaming cascades to all records.

    \item[\cmd{/delete} \param{challenge\_id}]
    Purge with recursive point deduction and hint refund.

    \item[\cmd{/list}]
    Show all challenges with Posted/Draft status.

    \item[\cmd{/show} \param{challenge\_id}]
    Reveal hidden flag text and metadata.

    \item[\cmd{/add\_hint} \param{challenge\_id text cost}]
    Attach a purchasable hint. Hint button appears on live embed automatically.

    \item[\cmd{/remove\_hint} \param{hint\_id}]
    Remove hint with automatic refunds to all buyers.
\end{description}

\section{Player Management}

\begin{description}[leftmargin=0pt, itemsep=0.4em]
    \item[\cmd{/revoke} \param{member challenge\_id}]
    Remove a solve. Deducts base points plus any first-blood bonus earned at that solve position.

    \item[\cmd{/ban\_user} \param{member}]
    Block all flag submissions and hint purchases.

    \item[\cmd{/unban\_user} \param{member}]
    Restore full access.
\end{description}

\section{System \& Maintenance}

\begin{description}[leftmargin=0pt, itemsep=0.4em]
    \item[\cmd{/setup} [\param{leaderboard\_channel challenge\_logs wrong\_submissions general\_channel champion\_role}]]
    Link channels and set champion role. Non-destructive and additive.

    \item[\cmd{/set\_rank\_role} \param{role points}]
    Define an auto-promotion milestone.

    \item[\cmd{/remove\_rank\_role} \param{role}]
    Remove a milestone from the promotion engine.

    \item[\cmd{/list\_rank\_roles}]
    Display all configured thresholds.

    \item[\cmd{/export}]
    Upload \texttt{bot.db} as an ephemeral Discord attachment for backup.

    \item[\cmd{/import} \param{file}]
    Live-swap the active database. Zero downtime.

    \item[\cmd{/reset\_config}]
    Wipe channel and role config only. Player data preserved.
\end{description}

\warning{
    \cmd{/wipe\_all} deletes \textbf{everything}: players, scores, challenges, hints, banlist,
    config, and all files in \texttt{/uploads/}. Irreversible. Always run \cmd{/export} first.
}
